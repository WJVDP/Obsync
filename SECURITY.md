# Security Policy

## Supported Versions

Security fixes are applied to the latest `main` branch and most recent release.

## Reporting a Vulnerability

Please do not report security issues in public GitHub issues.

Use one of these channels:

1. Open a private report via GitHub Security Advisories for this repository.
2. If advisories are unavailable, open a private maintainer contact request via GitHub and label it `security`.

Include:

- Affected component and version/commit
- Reproduction steps or proof of concept
- Impact assessment
- Any suggested mitigation

## Response Targets

- Initial acknowledgment: within 72 hours
- Triage decision: within 7 calendar days
- Remediation plan or fix timeline: within 30 calendar days

If we need more time, we will provide status updates at least weekly until resolution.

## Disclosure Process

- We follow coordinated disclosure.
- Please allow time for validation and patch preparation before public disclosure.
- After a fix is available, we will publish release notes with mitigation guidance.
